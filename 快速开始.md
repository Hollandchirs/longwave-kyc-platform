# KYC平台 - 快速开始指南

## 🚀 启动项目

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发服务器

```bash
npm run dev
```

服务器会在 `http://localhost:5173` 启动

### 3. 构建生产版本

```bash
npm run build
```

构建后的文件会在 `dist/` 目录

### 4. 预览生产版本

```bash
npm run preview
```

## 🎯 演示流程

1. **首页** → 展示三个主要服务选项
2. **商业注册** → KYC信息 → 填写董事/股东表单 → 提交
3. **返回首页** → 进入企业服务
4. **企业服务** → 上传PDF文件 → 提交
5. **返回首页** → 进入合规中心
6. **合规中心** → 查看新加坡法规信息 → 返回首页

## 🎨 技术栈

- **框架**：React 18 + Vite
- **样式**：Tailwind CSS
- **路由**：React Router v6
- **图标**：Lucide React
- **状态管理**：React Hooks (useState)

## 📝 注意事项

### MVP版本特性
- ✅ 前端界面完整
- ✅ 所有交互流程可用
- ✅ 响应式设计
- ⚠️ **无后端**：数据不会保存
- ⚠️ **无数据库**：刷新页面后数据清空
- ⚠️ **PDF上传**：仅前端显示，未实际上传
- ⚠️ **表单验证**：仅客户端验证

### 适用场景
- 黑客松演示 ✅
- 概念验证 ✅
- 投资人展示 ✅
- 用户调研 ✅
- 生产环境 ❌（需要添加后端）

## 🎤 演示技巧

### 准备工作
1. 确保开发服务器在运行
2. 浏览器打开 `http://localhost:5173`
3. 准备一个PDF文件用于演示上传
4. 测试所有页面跳转

### 演示要点
1. **强调痛点**：信息不对称、沟通碎片化
2. **展示完整流程**：从KYC到合规的完整旅程
3. **突出用户体验**：流畅的交互、清晰的界面
4. **说明扩展性**：易于添加新国家、新功能
5. **提及商业模式**：订阅费用、交易费用

### 常见问题准备
- ❓ **数据如何保存？** → 生产版本会有后端数据库
- ❓ **KYC如何验证？** → 接入Oracle等数据供应商API
- ❓ **支持哪些国家？** → 目前新加坡，计划扩展到香港、迪拜等
- ❓ **如何盈利？** → 服务商订阅费 + 每笔交易费用
- ❓ **数据安全？** → 端到端加密、SOC 2合规
